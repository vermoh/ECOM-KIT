# User Flows

## Flow 1. Вход в платформу
1. Пользователь открывает Control Plane.
2. Вводит логин и пароль.
3. Система проверяет учетные данные.
4. Система определяет membership и доступные организации.
5. Пользователь выбирает организацию, если их несколько.
6. Система показывает доступные сервисы в рамках tenant и effective permissions.

## Flow 2. Создание пользователя в организации
1. Admin открывает Users.
2. Создаёт пользователя.
3. Назначает роль.
4. Указывает срок доступа.
5. Указывает доступные сервисы.
6. Система создаёт membership, access grant и service access.
7. Событие логируется.

## Flow 3. Настройка AI provider
1. Admin открывает настройки AI provider.
2. Вводит OpenRouter API key.
3. Указывает scope: organization-level или user-level.
4. Система валидирует формат.
5. Система шифрует и сохраняет ключ.
6. Событие логируется.

## Flow 4. Создание CSV проекта
1. Service user открывает CSV сервис.
2. Создаёт project.
3. Загружает CSV.
4. Система запускает validation.
5. Если validation успешна — создаётся UploadJob со статусом validated.
6. Пользователь переходит к этапу schema generation.

## Flow 5. Генерация и утверждение схемы
1. Пользователь запускает schema generation.
2. Система анализирует названия товаров, category и subcategory.
3. Система строит SchemaTemplate.
4. Пользователь просматривает поля, edit / delete / add при необходимости.
5. Пользователь утверждает шаблон.
6. Статус SchemaTemplate = approved.

## Flow 6. Enrichment
1. Пользователь запускает enrichment.
2. Система создаёт EnrichmentRun.
3. Worker обрабатывает строки асинхронно.
4. По строкам формируются значения характеристик и confidence.
5. При неоднозначности создаются Collisions.
6. После завершения статус run обновляется.

## Flow 7. Разбор коллизий
1. Reviewer открывает очередь коллизий.
2. Видит проблемные записи и предложенные варианты.
3. Принимает решение вручную.
4. Система сохраняет resolution и логирует действие.
5. После закрытия всех обязательных коллизий экспорт разблокируется.

## Flow 8. Экспорт
1. Пользователь запускает ExportJob.
2. Система собирает данные, сериализует итоговый CSV.
3. Формирует ссылку на скачивание или запись в object storage.
4. Логирует экспорт.

## Flow 9. Истечение доступа
1. AccessGrant пользователя истекает.
2. Пользователь теряет возможность входа в сервис или выполнения новых мутаций.
3. Активные долгие операции завершаются по политике revocation.
4. Событие логируется.
