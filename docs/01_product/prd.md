# Product Requirements Document

## Название
AI-driven CSV Enrichment Platform

## Цель продукта
Предоставить B2B-пользователям управляемый инструмент для преобразования сырых товарных CSV в структурированные, обогащённые данные с характеристиками, пригодные для обратного импорта в e-commerce системы.

## Целевые пользователи
- менеджеры контента
- категорийные менеджеры
- e-commerce операторы
- marketplace / catalog managers
- администраторы организаций
- суперадминистраторы платформы

## Основные боли пользователей
- исходные CSV неполные и неструктурированные
- характеристики товаров отсутствуют или не унифицированы
- ручное заполнение слишком долгое и дорогое
- пользователи не доверяют полностью автоматическому AI-процессу без контроля
- разные организации и сотрудники должны быть строго изолированы
- доступ к сервисам и AI-ключам должен управляться централизованно

## Основной пользовательский результат
Пользователь получает:
1. проверенный шаблон характеристик,
2. заполненные характеристики по SKU,
3. список коллизий для ручной проверки,
4. итоговый CSV для обратного импорта,
5. опционально SEO-описания.

## MVP Scope
- Control Plane
- organizations
- users
- roles
- access grants
- service enablement
- OpenRouter key config
- CSV project creation
- upload + validation
- schema generation
- schema approval
- enrichment run
- collision review
- CSV export

## Не входит в MVP
- полноценный биллинг
- versioned prompts marketplace
- интеграции с внешними PIM/ERP
- multi-language SEO orchestration
- self-serve pricing portal
- advanced analytics across tenants

## Ключевые user stories

### Platform admin
- как super_admin, я хочу создавать организации, чтобы подключать новых клиентов;
- как super_admin, я хочу включать и отключать моносервисы, чтобы управлять доступностью функций;
- как super_admin, я хочу видеть аудит действий, чтобы расследовать проблемы.

### Organization admin
- как admin организации, я хочу создавать пользователей внутри своей организации;
- как admin организации, я хочу ограничивать срок доступа пользователя;
- как admin организации, я хочу назначать доступ к конкретному сервису;
- как admin организации, я хочу задать AI provider key для своей организации.

### Service user
- как service_user, я хочу загрузить CSV, чтобы начать обработку;
- как service_user, я хочу увидеть предложенный шаблон характеристик, чтобы утвердить его;
- как service_user, я хочу запустить enrichment после утверждения шаблона;
- как reviewer, я хочу разобрать коллизии вручную;
- как operator, я хочу выгрузить итоговый CSV.

## Success Criteria
- пользователь может пройти путь upload → schema approval → enrichment → review → export без обращения к разработчику;
- ни один пользователь не может получить доступ к данным другой организации;
- сервис недоступен пользователю после истечения срока доступа;
- тяжёлые задачи выполняются асинхронно и могут быть восстановлены после сбоя;
- все критические действия записываются в audit log.

## Product Principles
- контроль важнее полной автоматизации;
- прозрачность важнее “магии”;
- структура важнее красивой демонстрации;
- ошибки не должны разрушать проект;
- одно неверное AI-решение не должно портить весь набор данных.
