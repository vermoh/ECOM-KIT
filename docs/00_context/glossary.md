# Glossary

## Общие термины

**Control Plane**  
Управляющий слой платформы. Отвечает за аутентификацию, авторизацию, организации, пользователей, доступ к сервисам, конфигурацию AI-провайдеров и аудит.

**Service Plane**  
Набор предметных моносервисов, подключённых к Control Plane. Каждый сервис реализует свою бизнес-функцию и работает в tenant-scoped режиме.

**Tenant**  
Организация в системе. Базовая единица изоляции данных и прав доступа.

**Tenant Isolation**  
Принцип, по которому данные, права и операции одной организации не должны быть доступны другой организации без явно предусмотренного механизма.

**Monoservice / Моносервис**  
Отдельный прикладной сервис с собственной предметной логикой, UI, API и pipeline, подключаемый к общей платформе через Control Plane.

**RBAC (Role-Based Access Control)**  
Модель доступа на основе ролей.

**ABAC (Attribute-Based Access Control)**  
Модель доступа на основе атрибутов, например organization_id, access_expiry, ownership, service_enabled.

**Access Grant**  
Запись, определяющая право пользователя на доступ к организации, сервису или набору функций, включая срок действия.

**Provider Config**  
Конфигурация AI-провайдера, включая ключ, модель, лимиты, параметры timeout и retry.

**Project**  
Рабочая единица в CSV Enrichment Service, объединяющая загрузку файла, схему характеристик, enrichment run, collisions и экспорт.

**Schema Template**  
Шаблон структуры характеристик, сгенерированный на основе категорий и названий товаров до массового enrichment.

**Schema Field**  
Отдельная характеристика в шаблоне: название, тип, признак filterable, unit, allowed values и правила заполнения.

**Enrichment Run**  
Конкретный запуск процесса обогащения данных.

**Collision**  
Коллизия или конфликт, когда система не может с достаточной уверенностью определить значение характеристики, обнаруживает неоднозначность, противоречие или нарушение схемы.

**Review Task**  
Задача на ручную проверку человеком.

**Export Job**  
Задача на сбор и экспорт итогового результата в CSV.

**Human-in-the-loop**  
Подход, при котором критические или неоднозначные этапы процесса требуют участия человека.

**Source of Truth**  
Документ или сущность, считающиеся приоритетным и авторитетным источником истины для конкретной области.

## Технические термины

**JWT (JSON Web Token)**  
Токен аутентификации, передающий claims о пользователе, tenant и правах.

**Claim**  
Поле внутри токена, содержащее идентификаторы и права.

**Queue**  
Механизм асинхронной обработки длительных задач.

**Worker**  
Процесс, который исполняет фоновые задания из очереди.

**Idempotency**  
Свойство операции давать тот же результат при повторном вызове без нежелательных побочных эффектов.

**Observability**  
Наблюдаемость системы: логи, метрики, трассировка, диагностика.

**Dead Letter Queue**  
Очередь заданий, которые не удалось обработать после допустимого количества попыток.

**State Machine**  
Модель допустимых состояний сущности и переходов между ними.

**ADR (Architecture Decision Record)**  
Зафиксированное архитектурное решение с контекстом, причиной и последствиями.

**TDD (Technical Design Document)**  
Технический документ, описывающий архитектуру, ответственность компонентов, потоки данных и ограничения.

**NFR (Non-Functional Requirements)**  
Нефункциональные требования: производительность, надёжность, безопасность, масштабируемость и др.

**C4 Model**  
Подход к описанию архитектуры через уровни Context, Container, Component и Code.
